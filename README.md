# Лабораторная работа 2, Классы ядра и модели

# Student Class

## Описание

Класс `Student` предназначен для хранения информации о студенте. Он позволяет создавать объекты с различными контактными данными, включая телефон, телеграм, электронную почту и Git URL. Класс включает в себя валидацию введенных данных для обеспечения их корректности.

## Пример использования

```ruby
require_relative 'student_class'

begin
    student1 = Student.new(surname: "Иванов", name: "Иван", patronymic: "Иванович", phone: "+1 234-567-8901", git: "https://github.com/ivan/repo.git")
    puts student1

    student2 = Student.new(surname: "Петров", name: "Петр")
    puts student2

    student3 = Student.new(surname: "Сидоров", name: "Сидор", patronymic: "Сидорович", phone: "1234567890")
    puts student3

    student4 = Student.new(surname: "Кузнецов", name: "Сергей", git: "https://github.com/serezha/repo.git")
    puts student4

rescue ArgumentError => e
    puts e.message
end
```

## Методы

### `initialize(args = {})`

Конструктор класса, который принимает хэш с параметрами:

- `:surname` - Фамилия (обязательное поле)
- `:name` - Имя (обязательное поле)
- `:patronymic` - Отчество (необязательное поле)
- `:phone` - Телефон (необязательное поле)
- `:telegram` - Телеграм (необязательное поле)
- `:email` - Электронная почта (необязательное поле)
- `:git` - Git URL (необязательное поле)

### `validate_phone(phone)`

Метод класса для проверки корректности телефонного номера.

### `validate_email(email)`

Метод класса для проверки корректности электронной почты.

### `validate_telegram(telegram)`

Метод класса для проверки корректности телеграм-имени.

### `validate_git(git)`

Метод класса для проверки корректности Git URL.

### `set_contacts(args = {})`

Метод для установки значений контактных данных. Принимает хэш с необязательными полями.

### `to_s`

Метод для вывода информации о студенте в виде строки.

### `validate`

Метод для проверки наличия Git URL и контактной информации. Выбрасывает `ArgumentError`, если данные некорректны.

## Исключения

Класс выбрасывает `ArgumentError` в следующих случаях:

- Неправильно введен телефон
- Неправильно введен email
- Неправильно введен git
- Неправильно введен телеграм
- Отсутствует Git URL
- Отсутствует контактная информация
